<objective>
Hook the plugin's `onunload` lifecycle to clear `data.tokens` so disabling the plugin (Settings → Community plugins → toggle off) does not leave OAuth access/refresh tokens at rest in `data.json`. Conservative scope: only `data.tokens` is cleared. Sync state (`manifest`, `auditRing`, `lastSince`, `ignoredStaleIds`, `pendingServerDeletes`) is preserved so re-enabling resumes the user's view without forcing a full re-pull.

Purpose: closes a security smell in advance of first real users. Today's `onunload` only stops polling — tokens persist indefinitely in `data.json` even after the user disables the plugin. From the user's mental model, "off" should mean "not signed in".
</objective>

  <behavior>
    On plugin disable (Obsidian calls `onunload`), the plugin must:
    1. Stop the polling interval (existing behavior — preserve).
    2. If `data.tokens !== null`, clear it (`this.data.tokens = null`).
    3. Persist via `this.saveAll()` so the cleared state hits disk before the plugin instance is destroyed.
    4. Be tolerant of the case where tokens were already null (no extra writes, no errors).

    `onunload` is synchronous in the Obsidian Plugin API but Obsidian awaits the promise if `onunload` returns one. Make the body async so we can `await this.saveAll()`. The Plugin API supports this — see the Obsidian docs ("returning a promise from onunload"). If for some reason a fire-and-forget pattern is preferred, document the rationale in a comment above the call.

    Do NOT clear `manifest`, `auditRing`, `lastSince`, `ignoredStaleIds`, or `pendingServerDeletes`. Re-enable should resume from the user's last-known state, not force a full re-pull. The existing `Reset local sync state` palette command (in `src/main.ts:467`) is the path to wipe everything.

    Edge case — startup blocked: if `this.startupBlocked === true`, the plugin loaded but never signed in beyond startup. Tokens should still be null in that case, but defensively call the same clear path so the disable contract is uniform regardless of state.
  </behavior>
  <action>
    Edit `src/main.ts` `onunload`:

    ```typescript
    async onunload(): Promise<void> {
        this.stopPolling();
        // Conservative: clear only OAuth tokens so disabling the plugin
        // does not leave credentials at rest in data.json. Sync state
        // (manifest, audit ring, lastSince, ignored ids, pending server
        // deletes) is preserved so re-enable resumes without a full
        // re-pull. The Reset local sync state command wipes the rest.
        if (this.data.tokens !== null) {
            this.data.tokens = null;
            await this.saveAll();
        }
    }
    ```
