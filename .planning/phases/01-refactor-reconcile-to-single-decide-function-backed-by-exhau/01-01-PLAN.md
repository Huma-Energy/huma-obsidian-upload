```typescript
// Pure decision for one UUID. Caller passes the three views (any may be null);
// caller is responsible for excluded-folder filtering, duplicate-uuid skipping,
// synthetic-server-entry promotion, AND server-side rename handling.
//
// Returns null when no action is required (in-sync, or impossible state).
// Returns one SyncAction; the rename-local + same-cycle pull/push case is
// handled by the dispatch emitting BOTH actions for the same UUID:
//   1. dispatch emits rename-local directly,
//   2. dispatch synthesizes scanShim = { ...scan, path: server.path } and
//      passes it to decide as the `scan` argument so decide naturally
//      produces any pull/push at the post-rename path.
//
// Conflict counting: when decide returns a `push` AND server.version > local.version
// AND scanned.hash !== local.hash, dispatch increments stats.conflict.
export function decide(
  server: ManifestEntry | null,
  local: ManifestRecord | null,
  scan: ScannedFile | null,
): SyncAction | null;
```

Server-side rename handling — BINDING CONTRACT (resolves the rename-local + pull/push pairing):
- decide is single-action with 3 args. NO 4th `logicalPath` parameter.
- Dispatch detects serverRenamed: `server && local && scan && server.deleted_at === null && server.path !== local.path && scan.path === local.path`.
- When serverRenamed:
    1. dispatch pushes a `rename-local` action directly (uses fromPath=local.path, toPath=server.path),
    2. dispatch rebinds `effectiveScan = { ...scan, path: server.path }` and passes that to decide,
    3. decide sees scan.path === server.path → no plugin-side rename branch fires; the body-change comparison decides pull vs push vs no-op naturally.
- This preserves the existing pattern (rename-local emitted alongside subsequent pull/push) without bloating decide's signature beyond the ROADMAP-locked 3 args.

        // Step A: filter excluded paths from all three views before any keying
        // or comparison. Files in excluded folders never push, never pull,
        // never go stale.

        );

        );

        );

        // UUID is a corruption case — refuse to act on it; surface for the
        // engine to alert.

        for (const id of localById.keys()) uuidUnion.add(id);
        for (const id of scannedByUuid.keys()) uuidUnion.add(id);
